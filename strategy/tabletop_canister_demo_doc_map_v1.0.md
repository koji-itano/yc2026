# Real Physical Gigs Tabletop Canister Demo Doc Map v1.0

## Scope

This page tells the team which demo document to use at each stage.

- Use it as the handoff page for anyone joining the demo late
- Keep it short and operational
- Pair it with `README.md`

## Recommended Order

| Stage | Primary doc | Why |
|---|---|---|
| 1. Understand the story | `strategy/tabletop_canister_demo_runbook_v1.0.md` | Defines the full tabletop-only flow from alert to payout |
| 2. Print physical materials | `strategy/tabletop_canister_label_pack_v1.0.md` | Gives one printable source for all required labels |
| 3. Pack and stage the desk | `strategy/tabletop_canister_onsite_asset_checklist_v1.0.md` | Covers props, owners, desk setup, and go/no-go checks |
| 4. Lock the UI wording | `strategy/tabletop_canister_demo_copy_sheet_v1.0.md` | Keeps laptop, phone, and overlay copy consistent |
| 5. Rehearse the live sequence | `strategy/tabletop_canister_demo_run_of_show_v1.0.md` | Assigns timed actions for 2-person and 3-person teams |
| 6. Run the judged demo | `strategy/tabletop_canister_demo_quick_card_v1.0.md` | Gives the shortest live reference during the take |
| 7. Reset between takes | `strategy/tabletop_canister_onsite_asset_checklist_v1.0.md` | Fast reset steps are already included there |

## If You Only Read Two Docs

| Situation | Read these two docs |
|---|---|
| You are the presenter | `strategy/tabletop_canister_demo_quick_card_v1.0.md` and `strategy/tabletop_canister_demo_run_of_show_v1.0.md` |
| You are the phone operator | `strategy/tabletop_canister_demo_quick_card_v1.0.md` and `strategy/tabletop_canister_onsite_asset_checklist_v1.0.md` |
| You are setting up the physical table | `strategy/tabletop_canister_label_pack_v1.0.md` and `strategy/tabletop_canister_onsite_asset_checklist_v1.0.md` |
| You are implementing UI states | `strategy/tabletop_canister_demo_copy_sheet_v1.0.md` and `strategy/tabletop_canister_demo_runbook_v1.0.md` |

## Minimal Live Stack

If time is short, the minimum useful stack is:

1. `strategy/tabletop_canister_demo_runbook_v1.0.md`
2. `strategy/tabletop_canister_onsite_asset_checklist_v1.0.md`
3. `strategy/tabletop_canister_demo_quick_card_v1.0.md`

## Done Condition

The doc set is usable when a new teammate can look at this page, grab the right artifact in under 30 seconds, and run their part of the tabletop demo without hunting across the repo.
